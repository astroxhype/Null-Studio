◆ How it works

One bit per channel, per pixel.

Your message is encoded into the least significant bit of each red, green, blue channel. A 1920×1080 image can carry ~777KB of hidden payload with no visible change.

◆ When to use a password

Defense against detection.

Even with steganalysis, XOR-encrypted content resembles statistical noise. The same password must be used to reveal.

◆ How signing works

Hash, sign, embed, done.

The image's SHA-256 is signed with your ECDSA-P256 private key. The signature, your public key, a timestamp, and an optional identity string are packaged as XMP metadata.

◆ Tamper evidence

One byte breaks it all.

A verifier recomputes the hash and checks the signature. Any pixel edit, any metadata tweak, any format conversion after signing = seal broken. Cryptographically definite.

◆ What we attempt

Every signature, every chunk.

We scan for JPEG markers (FFD8…FFD9), PNG chunks (IHDR/IDAT/IEND), HEIC boxes (ftyp/mdat), EXIF thumbnails embedded as JFIF-within-JFIF, ICC profiles, and XMP packets.
